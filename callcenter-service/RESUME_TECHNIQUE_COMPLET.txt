  ============================================================
  3.1 ENTITÉS (base de données call-center)
  ============================================================

  ── Request (table principale) ──
  PK      : idR (Long, auto)
  Champs  : title, description, note, attachmentPath, deadline (LocalDate),
            updatedAt, createdAt
  Enums   : status (Status), requestType (RequestType),
            categoryRequest (CategoryRequest), priority (Priority)
  Relations:
    • @ManyToOne  → User user (créateur, FK user_id)
    • @ManyToOne  → User agent (agent assigné, FK agent_id)
    • @ManyToMany → List<Question> questions (table request_questions)
    • @OneToOne   → Report report
    • @OneToMany  → List<Logs> logs (cascade ALL, orphanRemoval)
    • @OneToMany  → List<Submission> submissionList (cascade ALL, orphanRemoval)

  ── User ──
  PK      : idUser (Long, auto)
  Champs  : fullName, email, username, enabled (boolean)
  Enum    : role (Role)
  Transient: getFirstName(), getLastName() (split de fullName)
  Relations:
    • @OneToMany → List<Request> requests (cascade ALL)
    • @OneToMany → Set<Request> assignedRequests (pas de cascade)
    • @OneToMany → Set<AgentLeave> leaves (cascade ALL)
    • @OneToMany → List<Notification> notifications (pas de cascade)

  ── Report ──
  PK      : id (Long, auto)
  Champs  : requestTitle, requestType, generatedDate, status (ReportStatus),
            approvedDate, sentDate, totalContacts, contactedContacts,
            contactRate, statisticsData (JSON LONGTEXT), pdfPath,
            aiInsightsData (JSON LONGTEXT), aiGeneratedDate
  Relation: @OneToOne(mappedBy="report") → Request

  ── Question ──
  PK      : id (Long, auto)
  Champs  : text, questionType (QuestionType), options (List<String>)
  Relation: @ManyToMany(mappedBy="questions") → Set<Request>

  ── Response ──
  PK      : id (Long, auto)
  Champs  : answer, multiAnswer (List<String>), booleanAnswer, numberAnswer,
            dateAnswer (LocalDate), timeAnswer (LocalTime)
  Relations:
    • @ManyToOne → Submission
    • @ManyToOne → Question

  ── Submission ──
  PK      : id (Long, auto)
  Champs  : submissionDate (LocalDate), contactId (Long)
  Relations:
    • @ManyToOne → Request
    • @OneToMany → List<Response> responses

  ── Notification ──
  PK      : id (Long, auto)
  Champs  : message, type (NotificationType), timestamp, isRead (boolean)
  Relation: @ManyToOne → User agent

  ── Logs ──
  PK      : id (Long, auto)
  Champs  : logAction (LogAction), actionDescription, details (TEXT),
            oldStatus/newStatus, oldPriority/newPriority,
            oldAssignedAgent/newAssignedAgent, timestamp, performedByUserId,
            performedByUserName, ipAddress, userAgent
  Relation: @ManyToOne → Request

  ── AgentLeave ──
  PK      : id (Long, auto)
  Champs  : startDate, endDate (LocalDate), status (String)
  Relation: @ManyToOne → User agent

  ── RequestContactStatus ──
  PK      : id (Long, auto)
  Champs  : contactId, status (ContactStatus), callNote (TEXT),
            lastCallAttempt, createdAt, updatedAt
  Relation: @ManyToOne → Request

  ── PasswordResetToken ──
  PK      : id (Long, auto)
  Champs  : token (unique), email, expiryDate, used (boolean)

9. SÉCURITÉ
================================================================================

  • SSO via Keycloak (OAuth2 / OpenID Connect)
  • JWT Bearer Token sur toutes les requêtes API
  • Validation JWT au niveau du Gateway ET du callcenter-service
  • 4 rôles : ADMIN, MANAGER, AGENT, SURVEY_REQUESTER
  • CORS configuré au Gateway (origine localhost:4200)
  • Contact-service : sécurité désactivée (protégé par le gateway)
  • WebSocket : endpoint /ws/** est public (auth via topic)
  • Endpoints publics : forgot-password, reset-password, swagger
